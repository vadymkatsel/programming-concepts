📝 Lists Cheat Sheet

Тут зібрані основні операції, які знадобляться вам для виконання завдань.

🔹 1. Створення та доступ

Операція Синтаксис Опис
Створення lst = [1, 2, 3] Створити список із даними
Пустий список lst = [] Ініціалізація пустим списком
Перший елемент lst[0] Індексація починається з 0
Останній елемент lst[-1] Негативний індекс рахує з кінця
Довжина len(lst) Кількість елементів у списку

🔹 2. Зрізи (Slicing)

Загальний синтаксис: list[start:stop:step]

arr = [10, 20, 30, 40, 50]

arr[1:4]   # [20, 30, 40] - від індексу 1 до 3 включно
arr[:3]    # [10, 20, 30] - перші 3 елементи
arr[::2]   # [10, 30, 50] - кожен другий елемент
arr[::-1]  # [50, 40, 30, 20, 10] - розворот списку

🔹 3. Додавання та зміна

nums = [1, 2, 3]

nums.append(4)    # [1, 2, 3, 4] -> Додає елемент в кінець (O(1))
nums[0] = 100     # [100, 2, 3, 4] -> Змінює існуючий елемент

🔹 4. Ітерація (Цикли).

Два основні способи пробігтися по списку: - Коли потрібен тільки елемент

for item in nums:
    print(item)
  • Коли потрібен індекс (наприклад, для формул \(y_{i+1} - y_i\)):
for i in range(len(nums)):
    print(f"Index: {i}, Value: {nums[i]}")
  • Коли потрібно і те, і інше (Pythonic way):
for i, val in enumerate(nums):
    print(f"Index: {i}, Value: {val}")

👨‍💻 Live Coding 1: Два вказівники

Контекст: Ми працюємо над розробкою бази даних. У нас є список заповнений певними значеннями, але от халепа! Виявилось, що значення в ньому знаходяться в оберненому порядку. Нам потрібно “перевернути” значення в списку, при тому не створюючи нового списку.

Вхідні дані: список arr = [50, 40, 30, 20, 10]

Вихідні дані: обернений список arr = [10, 20, 30, 40, 50]

ПопередженняОбмеження

Не можна використовувати list slicing, оскільки це створить новий список. Маємо користуватись тільки циклами та модифікацією поточного списку

🛠 Guided Practice: Кумулятивна сума

Контекст: Маємо список із чисел від 1 до 5. Нам потрібно знайти список часткових сум цих чисел за наступною формулою:

\(b_i = \sum_{k=0}^{i} a_k\)

або рекурентно:

\(b_i = b_{i-1} + a_i\).

Вхідні дані: arr = [1, 2, 3, 4, 5]

Вихідні дані: arr = [1, 3, 6, 10, 15]

👨‍💻 Live Coding 2: Скінченна різниця

Контекст: При роботі з дискретними даними (сигналами, часовими рядами) ми не можемо взяти аналітичну похідну. Натомість використовують чисельні методи. Найпростіший спосіб оцінити швидкість зміни функції — обчислити скінченні різниці.

У вас є дані з датчика положення робота, які записуються кожну секунду (дискретний час).

Масив positions містить координати в метрах.

Нам потрібно отримати масив швидкостей робота на кожному проміжку. Оскільки час між замірами \(\Delta t = 1c\), швидкість дорівнює просто різниці координат:

\[v_i = \frac{x_{i+1} - x_i}{\Delta t} = x_{i+1} - x_i\]

Вхідні дані: positions = [1, 4, 9, 16, 25, 36]

Вихідні дані: velocities = [3, 5, 7, 9, 11]

🛠 Guided Practice: Ковзне середнє

Контекст: Ми працюємо з даними, переданими з gps датчика. Під час перегляду даних, ми бачиму дивні цифри, які наврядчи відповідають реальності.

Ми хочемо побачити загальний тренд. Для цього використовують фільтр ковзного середнього. Ми замінюємо групу точок (вікно) на їх середнє арифметичне.

\(y_i = \frac{1}{k} \sum_{j=0}^{k-1} x_{i+j}\).

Вхідні дані: noisy_data = [10, 12, 100, 14, 11, 13] Розмір вікна \(k = 3\).

Вихідні дані: [40.66, 42.0, 41.66, 12.66]

💡 List Comprehension

У Python перед нами часто постає задача створення нових списків та заповнення нових даних: з тих даних, що ми вже маємо, або нових даних. Дотепер, нам була доступна опція створення списку за допомогою циклу. Припустимо, що нам потрібно створити список з квадратів парних чисел від 1 до 10:

squares = []
for i in range(10):
    if (i % 2 == 0):
        squares.append(i**2)

Цей підхід є непоганим і доволі логічним, проте аби ефективніше працювати зі списками, розробники Python придумали таку річ як List Comprehension. Вона дозволяє нам ефективно створювати нові списки всього в один рядок. Наступний фрагмент коду буде аналогічним до попереднього:

squares = [i**2 for i in range(10) if i % 2 == 0]

Загальна структура виглядає наступним чином:

[expression for item in iterable if condition]

[expression_if_true if condition else expression_if_false for item in iterable]

👨‍💻 Live Coding 3: Декартів добуток

Контекст: З метою проведення математичного дослідження, перед нами постала задача опису множини. У математиці ми зазвичай описуємо множини наступним чином: \(S = \{x^2 \mid x \in \mathbb{Z}, 0 \le x \le 10, x \text{ is even}\}\) або \(A \times B = \{(a, b) \mid a \in A, b \in B\}\).

Вхідні дані: Дві множини: A = [1, 2, 3], B = [4, 5]

Вихідні дані: - Декартів добуток \(A \times B\): [(1, 4), (1, 5), (2, 4), (2, 5), (3, 4), (3, 5)]

🛠 Guided Practice: трійки Піфагора

Контекст: Продовжуючи наше дослідження, нам потрібно також знайти всі цілочисельні трійки \((a, b, c)\), які задовольняють теорему Піфагора: \(a^2 + b^2 = c^2\).

Вхідні дані: Максимальне число \(N = 20\).

Вихідні дані: Список кортежів: [(3, 4, 5), (5, 12, 13), (6, 8, 10), (8, 15, 17), (9, 12, 15), (12, 16, 20)]

ПопередженняОбмеження

Щоб уникнути дублікатів (типу \(3^2+4^2=5^2\) і \(4^2+3^2=5^2\)), вводимо обмеження: \(1 \le a < b < c \le N\).

📐 Операції над матрицями, або вкладені списки

У Pyhon, як і в будь-якій іншій мові програмування, ми звичайно можемо представити матриці. В контексті сьогоднішнього заняття, нам буде зручно зробити це за допомогою вкладених списків:

matrix = [
    [1, 2],
    [3, 4],
    [5, 6]
]

👨‍💻 Live Coding 4: Транспонування

Контекст: Математики, особливо ті що працюють з лінійною алгеброю, часто використовують програмування для спрощення обрахунків і свого життя. Зараз нам потрібно буде виконати базову операцію (\(A^T\)) - транспонування матриці.

Вхідні дані: матриця \(3 \times 2\): matrix = [[1, 2], [3, 4], [5, 6]]

Вихідні дані: матриця \(2 \times 3\): transposed = [[1, 3, 5], [2, 4, 6]
